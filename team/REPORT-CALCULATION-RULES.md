# 日报 / 看板计算口径（已确认）

> 用途：防止后续日报与看板再次因取错列、错读表头而算错。
> 状态：已由土豆大人确认可按本口径执行。

## 一、通用输出字段

日报「业务速览」统一输出以下字段：

1. 业务
2. 进审量
3. 推审率
4. 大盘违规率
5. 状态

其中：
- **最新日期必须放最上面**
- **业务速览必须使用表格**

---

## 二、各类表的取数字段

### 1）表1类业务
适用业务：
- steam昵称简介
- 增量昵称简介
- 战绩昵称
- 融媒体短文本
- steam成就标题简介

字段口径：
- **进审量** = `总进审量`
- **推审率** = `人审推审率`
- **大盘违规率** = `大盘违规率`
- **数据差 / 链路异常值** = `数据量误差值`

补充说明：
- `steam成就标题简介` 也按 **表1同口径** 处理
- 若目标日期无数据，则按“无数据/断更”处理，不要瞎补算

---

### 2）表2普通手动表
适用业务示例：
- TapTap-头像-用户资料图片
- TapTap-融媒体长文本

字段口径：
- **进审量** = `总进审量`
- **推审率** = `推审率(手动)`
- **大盘违规率** = `违规率(手动)`

---

### 3）头像类表（含额外字段）
适用业务示例：
- steam头像封面
- 战绩头像

字段口径：
- **进审量** = `总进审量`
- **推审率** = `推审率(手动)`
- **大盘违规率** = `违规率(手动)`

注意：
- 不要把 `拦截率`、`机审通过中的黑样本` 误当成推审率或违规率
- 某些单元格可能出现公式或重复表头，取第一组有效表头即可

---

### 4）小镇照片类表
适用业务：
- 国内小镇照片
- 海外小镇照片

字段口径：
- **进审量** = 第2列（`总进审量` / 实际总量列）
- **推审率** = `初审推审率`
- **大盘违规率** = `总违规率`

注意：
- 不要把 `人审机审命中量` 当成推审率
- 不要把前几列编号 `1/2/3/4` 误当业务指标名

---

### 5）小镇书籍类表
适用业务：
- 国内小镇书籍
- 海外小镇书籍

字段口径：
- **进审量** = `总进审量`
- **推审率** = `推审率`
- **大盘违规率** = `违规率`

注意：
- `易盾命中准确率` 不是日报业务速览里的推审率/违规率

---

### 6）舆情类表
适用业务：
- 国内小镇舆情
- 海外小镇舆情

字段口径：
- **进审量** = `总进审量`
- **推审率** = `推审率(手动)`
- **大盘违规率** = `违规率(手动)`

注意：
- `云上命中准确率` 不是业务速览里的推审率/违规率
- 不要误取后半段重复表头或空白区

---

## 三、异常判断相关口径

### 数据差 / 链路异常
仅对 **表1类业务**，直接读取：
- `数据量误差值`

不要自行推导，不要拿其他列相减代替。

### 无数据 / 断更
如果目标日期没有该业务数据：
- 直接标记为 **无数据 / 疑似断更 / 数据未更新**
- 不要把上一日数据伪装成当日数据
- 若确需补位，必须明确写“最新可用日是 X 日”

---

## 四、执行规则

1. **先识别业务属于哪类表，再取字段**
2. **禁止硬编码列号跨表复用**，优先按表头名取值
3. **遇到重复表头，只取第一组有效表头**
4. **遇到公式/异常单元格，先确认实际数值含义，再入表**
5. **日报生成后，至少抽检以下业务再发：**
   - steam昵称简介
   - 战绩昵称
   - steam头像封面
   - 战绩头像
   - 国内小镇照片
   - 国内小镇舆情
6. **若抽检值与表内原值不一致，不发送，先修正**

---

## 五、这次明确纠正过的典型错误（留档）

以下属于已经出现过、以后必须避免的错误：

- 把 `steam头像封面` 的推审率/违规率取错
- 把 `战绩头像` 的推审率取错
- 把 `steam昵称简介` / `战绩昵称` / `融媒体短文本` 的数据差取错
- 在数据未完全确认前就直接写日报
