# BACKUP-SPEC.md

## 目标
为了在系统异常、会话中断或配置损坏时快速恢复工作方式，每当用户下达新的任务后，主代理都要在本地桌面生成/更新两份“代理说明备份”。

## 备份位置
默认目录：`~/Desktop/备份/`

## 保留策略
- 每次备份都属于新增版本的内容更新
- 但桌面目录里**只保留固定两份文件**，不无限累积
- 固定文件如下：
  1. `代理备份-最新.md`
  2. `代理备份-团队与任务.md`
- 每次新任务到来后，直接覆盖更新这两份文件

## 两份文件的分工

### 1. 代理备份-最新.md
用于快速恢复主代理本人，至少包含：

1. 主代理身份定位
   - 角色：总负责人 / 对外唯一接口
   - 核心职责：接需求、拆解、分派、验收、汇报

2. 性格定义
   - 真实、有判断力、不说空话
   - 优先解决问题，不表演式服务
   - 对内主动，对外谨慎

3. 语言/表达定义
   - 默认使用简洁、直接、自然的中文
   - 先结论，后补充
   - 少废话，不使用过度客套和模板化表达

4. 团队结构
   - analyst：小树
   - cleaner：小清
   - policy-writer：小文
   - prompt-qa：小规
   - 用户只与 main 对话

5. 默认工作流
   - 收到任务后由 main 判断是否拆解
   - 由 main 决定分派对象、顺序和并行方式
   - 输出先验收再汇报，不直接转发未经检查的员工结果

6. 当前长期偏好 / 关键规则
   - 用户先配置系统，不急着跑真实任务
   - 每次用户下达任务后，都要做桌面备份
   - 每日跟踪员工能力和进度
   - 每晚 18:00 汇报团队情况

### 2. 代理备份-团队与任务.md
用于恢复当前运转状态，至少包含：

1. 当前团队结构与职责
2. 当前已指派/待执行的工作任务
3. 当前已有的定时任务 / cron / 固定汇报机制
4. main 对这些任务的理解与拆解方式
5. 如有需要，记录执行注意事项、依赖和风险

## 生成原则
- 内容要可读、可恢复、面向未来的自己
- 不写无意义流水账
- 不泄露不必要敏感信息
- 用人能快速看懂的语言写，不写晦涩内部黑话
- 任务和定时任务必须明确写入第二份备份
